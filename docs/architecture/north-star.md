# North Star

## Positioning

- **Product:** myLearning
- **Category:** AI-Native Workforce Capability Platform
- **Primary outcome:** Workforce Readiness
- **Core idea:** myLearning builds, verifies, reinforces, and measures workforce capability. Readiness is the result of that loop, not the software category.

## Current State

myLearning currently supports the beginning of the loop: organizations can create, assign, prioritize, complete, and track short learning experiences. Verify, Ask, Updates, durable event evidence, and readiness measurement remain future work.

## Target Direction

The product should help each learner take the most valuable next action while giving organizations credible evidence that people can perform required work. AI should be embedded in trusted retrieval, reinforcement, and evidence interpretation rather than added as an unbounded chat layer.

The north-star test for a feature is:

> Does this help build, verify, reinforce, or measure a capability that matters to current work?

## Open Questions

- Which small set of indicators best represents Workforce Readiness for the first customer segment?
- At what level should readiness be reported: capability, learner, team, role, organization, or some combination?
- What evidence and recency thresholds are required before myLearning can present a readiness conclusion?
